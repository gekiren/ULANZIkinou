# ULANZI CREATIVE DECK プラグイン開発における絶対遵守ルール

本プロジェクト（ULANZI CREATIVE DECK 自作プラグイン開発）において、すべての開発およびAIエージェントは、以下のルールを完全に遵守しなければなりません。

---

## 1. Git ブランチ管理 ＆ GitHub 連携ルール

1. **直接コミット・プッシュの禁止**:
   - `master` ブランチで直接開発作業を行ったり、勝手に `origin/master` にプッシュすることは**厳禁**です。
2. **ブランチ運用サイクル**:
   - 新機能の実装、バグ修正、その他すべての作業は必ず **`staging` ブランチ**、または `feature/xxx` などの作業ブランチを作成して行います。
3. **本番マージの基準**:
   - ローカルでの実装およびシミュレーターによる動作確認が完了し、ユーザーより**「本番へのマージ・プッシュ」の明確な承認**を得られた後にのみ、変更を `master` ブランチへマージし、GitHub（`origin/master`）へプッシュします。

---

## 2. 実装計画の提示 ＆ 承認サイクル (Planning & Approval)

1. **実装プランの作成**:
   - コードの修正、新規ファイルの作成、またはコマンドを実行する前に、必ず具体的な変更内容をまとめた「実装計画（`implementation_plan.md`）」を作成し、ユーザーに提示して承認（Proceed）を得てください。
   - **実装計画（`implementation_plan.md`）は、必ず「日本語」で作成および出力してください。**
2. **検証結果の報告**:
   - 実装と動作確認が完了した後は、必ず「完了報告（`walkthrough.md`）」を更新し、どのような動作検証を行ったかをユーザーに報告します。

---

## 3. コードポータビリティ ＆ 絶対パスの排除

1. **相対パスの徹底**:
   - プラグインの内部コード（`manifest.json`、`app.js`、`app.html`、`inspector.html` 等）において、特定のPC環境に依存する絶対パス（例: `C:\Users\toshi\...`）を記述することは**厳格に禁止**します。
   - アイコン、ライブラリ（`libs/js/`、`libs/css/`）などのリソース参照は、すべてプラグインルート（`com.ulanzi.mycustomplugin.ulanziPlugin`）を基準とした**相対パス**で記述してください。
2. **外部ライブラリ参照**:
   - 共通ライブラリを読み込む際は、コピーされた `libs/` ディレクトリ内のファイルを正しく参照してください。

---

## 4. ローカル検証手順 (シミュレーターの実行)

1. **シミュレーターの使用**:
   - 実機（Ulanzi Studio）でテストする前に、必ず `C:\ULANZIkinou\KOUSIKI\UlanziDeckPlugin-SDK-main\UlanziDeckSimulator` のシミュレーターを使用して動作検証を行ってください。
2. **検証の流れ**:
   - 作成または修正したプラグインフォルダを `UlanziDeckSimulator/plugins/` にコピーします。
   - `UlanziDeckSimulator` フォルダで `npm start` を実行し、ブラウザで `http://127.0.0.1:39069` にアクセスします。
   - シミュレーター上で「Refresh Plugin List」を押し、追加したアクションをキーに配置し、 Property Inspector が正しく動作するか確認します。
   - Node.jsプラグインの場合は、バックグラウンドのメインサービス（`node path/to/app.js`）を別プロセスとして手動で起動した上で通信のテストを行います。

---

## 5. 次回の会話への引き継ぎルール (Handover Rule)

AIエージェントのコンテキスト肥大化による誤作動を防ぐため、設計完了時、実装完了時、またはデバッグの節目において、AIは自発的にセッションの切り替えを提案します。
- 移行時には、現在の会話フォルダ直下に引き継ぎサマリー（`handover_summary.md`）を作成してください。
- ユーザーが次回のチャットで即座に状況を読み込めるよう、以下のテンプレートをチャットの最後に出力してください：

```text
作業前にルートにある DEVELOPMENT_RULES.md を確認してください。
下記のファイルの内容を読み込んで、指示に従って進めてください。
file:///C:/Users/toshi/.gemini/antigravity/brain/[Conversation-ID]/handover_summary.md
```
